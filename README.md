# DevOps Lab: Контейнеризированное веб-приложение (Nginx + Flask + PostgreSQL)

Проект представляет собой демонстрационную микросервисную архитектуру, разработанную для отработки навыков контейнеризации, сетевого взаимодействия и администрирования баз данных.

## 🚀 Архитектура проекта

Инфраструктура состоит из четырех изолированных сервисов, объединенных в общую сеть через **Docker Compose**:

1.  **Nginx (Reverse Proxy):** Входная точка приложения. Принимает HTTP-запросы на порту 80 и перенаправляет их на Backend, скрывая внутреннюю структуру сети.
2.  **Flask Web-App:** Backend-приложение на Python, обрабатывающее бизнес-логику и взаимодействующее с базой данных.
3.  **PostgreSQL 15:** Реляционная база данных для хранения данных приложения. Использован механизм **Docker Volumes** для обеспечения персистентности (сохранности) данных при перезапуске контейнеров.
4.  **pgAdmin 4:** Инструмент с графическим интерфейсом для мониторинга и администрирования БД.

## 🛠 Стек технологий

* **Оркестрация:** Docker, Docker Compose
* **Web/Proxy:** Nginx
* **Backend:** Python 3.x, Flask, Psycopg2
* **Database:** PostgreSQL 15
* **OS:** Linux (Alpine/Debian в контейнерах)

## 🛡 Безопасность и администрирование

В проекте реализованы следующие инженерные решения:
* **Изоляция:** Сервисы Backend и DB не имеют открытых портов наружу, доступ к ним возможен только через Proxy или внутреннюю сеть Docker.
* **Environment Variables:** Конфиденциальные данные (пароли, настройки БД) вынесены в переменные окружения.
* **Health Checks:** Использование `depends_on` гарантирует правильный порядок запуска сервисов (база -> приложение -> прокси).

## 📋 Как запустить проект

1. Убедитесь, что у вас установлен **Docker** и **Docker Compose**.
2. Склонируйте репозиторий:
    ```bash
    git clone https://github.com/kirillgt33/devops-lab-project.git
    cd devops-lab-project
    ```
3. Настройка переменных окружения (`.env`):
    Проект использует переменные окружения для изоляции чувствительных данных от исходного кода. 
    В корне репозитория находится файл-шаблон `.env.example`. Перед запуском контейнеров обязательно создайте локальный файл `.env` на его основе:
    ```bash
    cp .env.example .env
    ```
    После этого откройте созданный файл `.env` и заполните переменные:
    * `SECRET_KEY` — секретный ключ для защиты сессий Flask.
    * `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` — имя базы данных, логин и пароль для PostgreSQL (Docker сам инициализирует БД с этими параметрами при первом старте).
    * `PGADMIN_DEFAULT_EMAIL`, `PGADMIN_DEFAULT_PASSWORD` — доступы для входа в панель управления pgAdmin.

4. Запустите сборку и старт контейнеров:
    ```bash
    docker-compose up --build -d
    ```
5. Доступ к приложению:
    * **Веб-приложение:** `http://localhost`
    * **Панель pgAdmin:** `http://localhost:8080`

## Troubleshooting (Решение частых проблем)

### Ошибка подключения бэкенда: `FATAL: password authentication failed for user`
* **Симптом:** Контейнер бэкенда (`flask_backend`) бесконечно пишет в логи `База еще не готова, ждем...` и падает с флагом `unhealthy`, хотя база данных горит как `healthy`.
* **Причина:** Официальный образ PostgreSQL инициализирует пользователя и пароль **только при самом первом запуске** (когда создается volume). Если вы изменили данные в `.env` позже, Postgres продолжит использовать старые закэшированные креды.
* **Решение:** Выполните полный сброс локальных волюмов проекта, чтобы база пересоздалась с новыми переменными:
  ```bash
  docker-compose down -v
  docker-compose up -d --build
  ```

### Зависание запуска: `dependency web-app failed to start: container is unhealthy`
* **Симптом:** Docker Compose отказывается запускать Nginx, выдавая ошибку сборки.
* **Причина:** В проекте настроена строгая оркестрация через healthcheck. Nginx не запустится, пока Flask-приложение не ответит успешным статусом, а Flask в свою очередь ждет готовности базы. Если один из сервисов упал из-за внутренней ошибки кода или конфигурации, вся цепочка блокируется.
* **Решение:** Проверьте логи конкретного упавшего сервиса, чтобы найти точную причину (например, синтаксический краш в Python):
  ```bash
  docker logs flask_backend
  ```
