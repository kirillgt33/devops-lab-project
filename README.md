# DevOps Lab: Контейнеризированное веб-приложение (Nginx + Flask + PostgreSQL)

Проект представляет собой демонстрационную микросервисную архитектуру, разработанную для отработки навыков контейнеризации, сетевого взаимодействия и администрирования баз данных.

## 🚀 Архитектура проекта

Инфраструктура состоит из четырех изолированных сервисов, объединенных в общую сеть через **Docker Compose**:

1.  **Nginx (Reverse Proxy):** Входная точка приложения. Принимает HTTP-запросы на порту 80 и перенаправляет их на Backend, скрывая внутреннюю структуру сети.
2.  **Flask Web-App:** Backend-приложение на Python, обрабатывающее бизнес-логику и взаимодействующее с базой данных.
3.  **PostgreSQL 15:** Реляционная база данных для хранения данных приложения. Использован механизм **Docker Volumes** для обеспечения персистентности (сохранности) данных при перезапуске контейнеров.
4.  **pgAdmin 4:** Инструмент с графическим интерфейсом для мониторинга и администрирования БД.

## 🛠 Стек технологий

* **Оркестрация:** Docker, Docker Compose
* **Web/Proxy:** Nginx
* **Backend:** Python 3.x, Flask, Psycopg2
* **Database:** PostgreSQL 15
* **OS:** Linux (Alpine/Debian в контейнерах)

## 📋 Как запустить проект

1.  Убедитесь, что у вас установлен **Docker** и **Docker Compose**.
2.  Склонируйте репозиторий:
    ```bash
    git clone [https://github.com/kirillgt33/devops-lab-project.git](https://github.com/kirillgt33/devops-lab-project.git)
    cd devops-lab-project
    ```
3.  Запустите сборку и старт контейнеров:
    ```bash
    docker-compose up --build -d
    ```
4.  Доступ к приложению:
    * **Веб-приложение:** `http://localhost`
    * **Панель pgAdmin:** `http://localhost:8080` (логин: `admin@admin.com`, пароль: `secret`)

## 🛡 Безопасность и администрирование

В проекте реализованы следующие инженерные решения:
* **Изоляция:** Сервисы Backend и DB не имеют открытых портов наружу, доступ к ним возможен только через Proxy или внутреннюю сеть Docker.
* **Environment Variables:** Конфиденциальные данные (пароли, настройки БД) вынесены в переменные окружения.
* **Health Checks:** Использование `depends_on` гарантирует правильный порядок запуска сервисов (база -> приложение -> прокси).
